# D'Flow 바이브코딩 경진대회 브레인스토밍

작성일: 2026-07-11  
대상 프로젝트: 동국씨엠 PI(Process Innovation) 관리 플랫폼 D'Flow

## 1. 결론

D'Flow는 이미 단순한 WBS 웹앱이 아니다. WBS·간트·대시보드·회의·회의록 RAG·실시간 주간업무·공식 PPT/Excel 자동화까지 갖춘 PI 프로젝트 운영 플랫폼이다.

현재 가장 큰 과제는 기능을 더 추가하는 것이 아니다. 심사위원에게 다음과 같이 인식되도록 제품의 중심 스토리를 바꾸는 것이다.

> D'Flow는 회의의 한 문장을 PI 실행과 현장 성과까지 연결하는 AI Control Tower다.

핵심 메시지:

> 회의가 끝나는 순간, PI 실행이 시작됩니다.

## 2. 현재 제품의 강점

### 실행 관리

- 계층형 WBS와 간트 차트
- 영업일·공휴일·가중치를 반영한 진척 자동 계산
- 계획 대비 실적, 달성률, 지연 상태, SPI, S-Curve
- 칸반 보드, 산출물 첨부, 변경 이력
- Excel WBS 가져오기·내보내기

### 협업·운영

- PMO와 팀별 권한 관리
- Supabase RLS와 서버 액션 이중 권한 검증
- 회의 일정·반복 회의·참석자
- 공지사항, 근태현황, 멤버 관리
- 실시간 공동편집 주간업무 시트
- 동국씨엠 공식 양식 기반 주간보고 PPT 자동 생성

### AI·지식화

- WBS 기반 DK Bot
- 회의록 보관함과 문서별/전체 회의록 RAG
- 정확한 수치는 결정형 도메인 함수에서 계산
- LLM 실패 시 결정형 답변으로 폴백
- Gemini/OpenAI 호환 제공자 구조
- 의미검색, 키워드 정확검색, 출처 기반 회의록 답변

- 린트 통과
- 2026-06-29부터 2026-07-11까지 365개 커밋

## 3. 현재 상태의 경진대회 리스크

현재 제품은 훌륭한 `System of Record`이지만, 아직 완전한 `System of Action & Value`는 아니다.

- AI가 주로 검색·질의·요약에 머물러 있다.
- 회의록·WBS·주간보고가 각각 존재하지만 실행 흐름으로 강하게 연결되지는 않는다.
- 측정 중심이 작업 공정률에 치우쳐 있다.
- PI의 최종 성과인 리드타임·원가·품질·재고·생산성·에너지 효과가 모델에 없다.
- WBS 선후행 관계가 없어 지연의 연쇄 영향과 회복 시나리오를 보여주기 어렵다.
- README는 여전히 초기 WBS MVP 설명에 머물러 있다.
- README에 설명된 데모 모드와 실제 코드의 상태가 일치하지 않는다.
- 일부 PMO 분석 컴포넌트(`ProgressMatrix`, `VarianceRanking`, `DelayAging`, `DataHygiene`)가 구현되어 있지만 현재 대시보드에서 활용되지 않는다.

따라서 본선에서는 근태·공지·멤버 같은 기능을 주인공으로 삼기보다, 이 기능들이 PI 실행을 뒷받침하는 운영 기반임을 보여주는 편이 좋다.

## 4. 추천 킬러 기능: Decision-to-Value Loop

```text
회의록·현장 이슈
  → AI가 결정·액션·리스크를 원문 근거와 함께 추출
  → PMO가 WBS 변경안을 검토·승인
  → 담당·기한·WBS·알림에 반영
  → 일정 및 KPI 영향을 결정형 엔진으로 계산
  → 회복 시나리오 비교
  → 공식 주간보고와 다음 회의 브리핑 생성
```

### 4.1 회의록 → 실행계획 AI

AI가 다음 내용을 제안한다.

- 결정사항
- 액션 아이템
- 담당자·담당팀·지원팀
- 완료 기한
- 관련 기존 WBS
- 신규 WBS가 필요한지 여부
- 신규 리스크
- 일정·담당·상태 변경안
- 원문 근거 문장

AI가 즉시 DB를 변경하지 않고, 변경 전·후 diff를 보여준 뒤 PMO 승인 후 반영한다. 승인 결과는 기존 변경 이력에 남긴다.

### 4.2 PI Digital Thread

최소한 다음 연결을 시각화한다.

```text
AS-IS 문제
  → 개선과제
  → WBS 작업
  → 산출물·회의 결정
  → 현장 KPI
  → 기대 편익·실현 편익
```

KPI는 처음부터 거대한 체계로 만들 필요가 없다.

- 기준값
- 목표값
- 현재값
- 단위
- 측정 주기
- 데이터 출처
- 연결된 WBS 또는 마일스톤
- 기대 편익과 실현 편익

예시:

> 수주정보 중복 입력 → ERP–MES 인터페이스 개선 → 통합 테스트 → 수작업 입력시간 감소

### 4.3 지연 전파·What-if

최소한의 선후행 관계를 추가해 다음 시나리오를 비교한다.

- 기준정보 정제 지연
- 통합 테스트 시작 지연
- UAT와 Go-live 영향
- KPI 효과 실현 지연
- 인력 추가 또는 업무 병렬화에 따른 회복 효과

숫자와 일정은 결정형 엔진이 계산하고, LLM은 결과를 자연어로 설명한다.

### 4.4 AI 주간보고 Agent

다음 데이터를 모아 주간보고 초안을 생성한다.

- WBS 실적 변화
- 지난주·이번 주 회의록
- 회의·공지·근태
- 지연과 마감 임박 작업
- 산출물 첨부
- 이전 주간보고

출력:

- 금주 실적
- 차주 계획
- 주요 이슈
- 대응방안
- 경영진 3줄 요약
- 문장별 근거 링크와 데이터 신선도
- 승인 후 동국 공식 PPT

## 6. 본선 데모 시나리오

### 4~5분 버전

1. **문제 제시**  
   엑셀 WBS, 회의록 파일, 주간보고 PPT가 분리되어 있어 결정사항이 실행으로 이어지지 않는 상황을 보여준다.

2. **PI Mission Control**  
   `ERP–MES 통합 테스트 위험`과 관련 마일스톤·KPI를 보여준다.

3. **회의록 업로드**  
   예: “예외 코드 37건 미정리, ERP 담당이 8월 20일까지 조치, MES 지원 필요.”

4. **AI 변경안 생성**  
   결정 1건, 액션 2건, 리스크 1건, WBS 변경 2건을 원문 근거와 함께 제안한다.

5. **PMO 승인**  
   WBS·칸반·담당·기한이 즉시 바뀌고 변경 이력이 생성된다.

6. **영향 시뮬레이션**  
   지원 인력 추가 또는 병렬화 시 예상 지연이 줄어드는 모습을 보여준다.

7. **성과 연결**  
   기준정보 오류율, 수작업 시간, 통합 테스트 리드타임 등 KPI의 기준값·목표값·현재값을 보여준다.

8. **결과 보고**  
   동국 공식 주간보고 PPT를 생성한다.

마무리 문장:

> D'Flow는 프로젝트를 기록하는 도구가 아닙니다. 회의의 한 문장을 실행과 현장 성과까지 연결하는 PI 운영체계입니다.

## 9. 공고가 나온 뒤 조정할 항목

## 10. 전체 화면 구성·UI/UX·디자인 진단

### 종합 평가

현재 프로젝트는 일반적인 사내 프로젝트 관리 시스템 기준으로는 완성도가 높다. 특히 WBS와 주간업무 화면은 실제 업무 생산성을 고려한 기능이 잘 구현되어 있다. 다만 경진대회 우승작 관점에서는 아직 ‘PI Control Tower’보다 ‘잘 만든 프로젝트 관리 시스템’으로 보일 가능성이 있다.

핵심 개선 방향은 화면을 더 화려하게 만드는 것이 아니라, 포털과 대시보드에서 다음 세 가지가 즉시 보이게 만드는 것이다.

1. 어디가 위험한가
2. 왜 위험한가
3. 지금 무엇을 해야 하는가

### 영역별 평가

| 영역 | 평가 | 판단 |
|---|---:|---|
| 디자인 시스템 | 4/5 | 색상·카드·버튼·모달 스타일의 일관성이 좋음 |
| WBS·주간업무 | 4.5/5 | 프로젝트의 가장 강력한 경쟁력 |
| 메뉴·정보구조 | 3.5/5 | 기본 구조는 좋지만 메뉴가 평면적이고 역할 구분이 약함 |
| 포털 | 2.5/5 | 프로젝트 목록은 깔끔하지만 포트폴리오 관제 기능이 부족함 |
| 대시보드 | 3/5 | 현황 파악은 가능하지만 다음 행동으로 연결되지 않음 |
| 모바일·반응형 | 2/5 | WBS·주간업무·달력은 데스크톱 중심 |
| 접근성 | 3/5 | 공통 모달은 우수하지만 Drawer·Popover 보완 필요 |
| 경진대회 차별성 | 2.5/5 | 철강 PI라는 정체성이 화면에서 충분히 드러나지 않음 |

### 잘 구성된 부분

- 크림색 배경, 청록색 브랜드 컬러, 어두운 사이드바가 일관되게 사용된다.
- 카드·버튼·입력창·상태 칩·모달·로딩·빈 화면이 공통 컴포넌트로 정리되어 있다.
- 전체 포털과 활성 프로젝트 내부를 구분한 2단계 메뉴 구조가 명확하다.
- WBS에 고정 열, 간트, 인라인 편집, 변경 이력, 첨부파일, 전체화면, 동시 접속자 등 높은 업무 생산성이 구현되어 있다.
- 주간업무에 실시간 협업, 다중 셀 선택, 복사·붙여넣기, Undo/Redo, 자동 저장, PPT 출력이 구현되어 있다.
- 공통 모달에 포커스 트랩, Escape 닫기, 포커스 복원이 적용되어 있다.
- 전역 포커스 표시, 모션 감소 설정, 빈 상태, 로딩 스켈레톤 등 기본 UX가 잘 준비되어 있다.

### 본선 전 반드시 수정할 항목

#### 1. 모바일 메뉴에서 주간업무 누락

데스크톱 사이드바에는 `weekly`가 있지만 모바일 메뉴에는 없다. 모바일에서 직접 진입할 수 없는 핵심 기능이므로 최우선으로 수정한다.

#### 2. 프로젝트 상태를 날짜만으로 판단하는 문제

현재는 종료일이 지나면 WBS가 미완료여도 프로젝트가 완료로 표시될 수 있다. 상태를 다음처럼 분리한다.

- 시작 전
- 정상 진행
- 위험
- 지연
- 실제 완료

PI 시스템에서는 상태의 신뢰도가 매우 중요하다. 날짜 기반 상태와 실제 WBS 완료율·위험도를 분리해 표시한다.

#### 3. `PageHero`와 로딩 화면 정합성

여러 페이지에서 설명·KPI·배지·액션을 전달하지만 실제 Hero에는 제목만 표시되는 구조가 있다. 또한 로딩 화면은 과거의 큰 Hero와 KPI 영역을 보여줘 실제 화면과 레이아웃 차이가 발생한다.

- 포털: 큰 Hero 유지
- 일반 프로젝트 페이지: 제목·핵심 액션·KPI 2~3개의 Compact Header
- WBS·주간업무: Hero를 최소화하고 작업 공간 확대
- 대시보드: 중복 제목 영역 제거

#### 4. 포털을 프로젝트 목록에서 PI 관제 화면으로 전환

프로젝트 카드에 다음 정보를 추가한다.

- 계획 대비 실제 진척률
- 지연 작업 수
- 프로젝트 위험도
- 다음 마일스톤
- 마지막 업데이트 시점
- 담당 조직
- 실적 미입력 여부

상단 KPI도 전체 프로젝트 수·Active ratio보다 위험 프로젝트, 지연 과제, 임박한 마일스톤, 실적 입력 필요, PI 성과 실현율 중심으로 재구성한다. 검색·위험도 필터·담당 조직 필터·위험순 정렬도 필요하다.

#### 5. 대시보드의 행동 연결 강화

대시보드의 신호 타일과 알림을 실제 작업으로 연결한다.

- 위험 타일 → 지연 작업이 필터링된 WBS 또는 Kanban
- 일정 편차 → 간트 타임라인
- 마일스톤 → 해당 WBS 상세
- 실적 미입력 → 바로 수정 가능한 목록
- AI 분석 결과 → 근거 데이터와 해당 행

이미 구현되어 있지만 화면에서 사용하지 않는 `ProgressMatrix`, `VarianceRanking`, `DelayAging`, `DataHygiene` 컴포넌트를 PMO 대시보드에 복원한다.

#### 6. 역할별 대시보드 프리셋

- 경영진: 계획 대비 실적, 위험, 마일스톤, PI 성과
- PMO: 지연 작업, 편차 순위, 장기 지연, 데이터 품질
- 팀원: 내 작업, 이번 주 할 일, 실적 입력 필요, 예정 회의

완전히 별도 화면을 만들기보다 같은 대시보드에서 역할별 프리셋을 제공하는 방식이 현실적이다.

### 권장 메뉴 구조

```text
전사 영역
├─ PI 포트폴리오
├─ My Work
└─ 통합 회의록

활성 프로젝트
├─ Mission Control
├─ 실행
│  ├─ WBS / Gantt
│  ├─ Kanban
│  └─ 주간업무
├─ 협업
│  ├─ 회의
│  └─ 공지사항
├─ 운영
│  ├─ 구성원
│  └─ 근태
└─ 관리
   └─ 프로젝트 설정
```

모바일에서는 활성 프로젝트 메뉴를 먼저 보여주고, 전체 프로젝트 목록은 검색 가능한 접이식 프로젝트 선택기로 제공한다.

### 반응형·접근성 보완

- WBS는 모바일에서 간트 중심이 아니라 요약 리스트 또는 카드형 화면으로 제공한다.
- 주간업무는 모바일 읽기 전용 카드 뷰 또는 ‘데스크톱 편집 권장’ 안내를 제공한다.
- 주간업무의 행 이동·삭제 버튼은 hover 전용이 아니라 `focus-within`과 더보기 메뉴로도 접근 가능하게 한다.
- 모바일 메뉴와 WBS 상세 Drawer에는 포커스 트랩·Escape·포커스 복원을 적용한다.
- 버튼이 3~4개인 모달 Footer는 작은 화면에서 줄바꿈 또는 세로 배치한다.
- 회의록 검색은 debounce, 검색 중 표시, 결과 수, 오류 상태를 제공한다.
- DkBot과 우측 하단 Toast가 겹치지 않도록 Toast 위치를 조정한다.
- 다크모드 WBS의 하드코딩된 밝은 배경은 디자인 토큰으로 교체한다.
- 9~11px 보조 문구는 발표용 화면에서 읽을 수 있도록 최소 크기와 대비를 높인다.

### 디자인·브랜딩 보완

- `D'Flow`, `DK Flow`, `Project Management System`, `DK Bot` 등 브랜드 표현을 하나로 통일한다.
- `Active ratio`, `Quick access`, `Project library` 등 한국어와 영어가 혼용되는 레이블을 용어집 기준으로 정리한다.
- 전체 디자인을 바꿀 필요는 없지만, 포털과 Mission Control에 철강 공정·설비·생산 흐름을 연상시키는 산업적 시각 언어를 추가한다.
- 로그인과 홈 화면의 메시지를 일반 PMS가 아니라 “동국씨엠 PI 실행 Control Tower”로 변경한다.

### UI/UX 개선 우선순위

1. 모바일 주간업무 누락 및 잘못된 프로젝트 상태 수정
2. `PageHero` 정책 확정과 로딩 화면 정합성 개선
3. 포털 카드에 진척·편차·위험·마일스톤 추가
4. 대시보드 위험 신호에 WBS·Kanban Deep Link 연결
5. PMO 분석 컴포넌트 복원 및 역할별 대시보드 적용
6. 모바일 Drawer·WBS·주간업무의 접근성 및 반응형 보완
7. 브랜드명·PI 카피·한영 레이블 통일
8. 실제 모바일·태블릿·프로젝터 환경에서 최종 시각 검증

이 진단의 최종 목표는 “예쁜 업무 시스템”이 아니라 “주의가 필요한 PI를 즉시 찾아 행동하게 하는 화면”이다.

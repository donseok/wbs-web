# 60초 본편 — 촬영·합성 대본

작성일 2026-07-21 · 짝 문서: `2026-07-21-영상-60초.html`

영상은 **HTML 프레임 + 실화면 삽입**의 하이브리드다. HTML을 전체화면으로 재생·녹화해 뼈대를 만들고,
검은 슬롯 두 곳에 따로 녹화한 앱 화면을 얹는다. 이 문서는 그 두 조각을 어떻게 찍고 붙이는지를 정한다.

## 0. 녹화 준비 (양쪽 공통)

| 항목 | 값 | 이유 |
|---|---|---|
| 해상도 | 1920×1080 | HTML 무대가 이 크기 고정. 다른 값으로 찍으면 슬롯에 얹을 때 리사이즈 손실 |
| 프레임 | 60fps 권장 (30도 가능) | 스크롤·타이핑 장면의 끊김 방지 |
| 데이터 | **시드 데모 프로젝트만** | 실명·실데이터가 한 프레임이라도 들어가면 회수 불가 (전략서 §8) |
| 브라우저 | 전체화면, 확대 100%, 북마크바 숨김 | 크롬 UI가 보이면 "화면 녹화"가 아니라 "내 컴퓨터 구경"이 된다 |
| 커서 | 강조 표시 ON | 클릭 지점이 안 보이면 무엇이 일어났는지 전달되지 않는다 |
| 알림 | 방해금지 모드 | 메신저 팝업 한 번에 재촬영 |

HTML 프레임 녹화: 파일을 브라우저로 열고 `H`(UI 숨김) → `Space`(재생) → 61초 후 자동 정지.

## 1. 슬롯 A — 또박또박 (본편 0:11~0:28 중 약 13초)

한 번에 이어 찍는다. 컷 편집으로 붙이면 "진짜 되는 건가" 의심을 산다.

| # | 화면 | 동작 | 길이 | 확인 포인트 |
|---|---|---|---|---|
| 1 | 회의 생성 → 라이브 | 녹음 시작 버튼 클릭 | 2초 | 경과시간이 도는 게 보일 것 |
| 2 | 라이브 전사 | 말하는 대로 자막이 쌓이는 구간 | 4초 | **말과 자막의 시차가 짧은 구간**을 고를 것 |
| 3 | 화자 패널 | 자동 분리된 화자에 이름 붙이기 | 3초 | 화자 색이 전사 본문까지 바뀌는 순간까지 |
| 4 | 종료 직후 회의록 | 핵심·결정·Action Item이 이미 채워진 상태 | 3초 | **Action Item 블록에서 멈출 것** — 다음 장면(D'Flow)과 이어지는 고리 |
| 5 | 인용 점프 | 요약 문장의 시각 마커 클릭 → 오디오 점프 | 2초 | 재생 헤드가 이동하는 게 보여야 "근거 있는 AI"가 전달된다 |

대사(녹음할 회의 내용)는 4~5번 문장으로 짧게. 예시 —
> "예외 코드 정리가 아직 안 끝났습니다. 김OO 책임이 8월 20일까지 맡아주시고, MES 쪽 지원이 필요합니다."

이 한 문장이 액션으로 바뀌고, 그게 다음 슬롯에서 WBS가 된다. **두 슬롯은 같은 문장으로 이어져야 한다.**

## 2. 슬롯 B — D'Flow (본편 0:28~0:45 중 약 13초)

| # | 화면 | 동작 | 길이 | 확인 포인트 |
|---|---|---|---|---|
| 1 | 회의록 → 액션 | 슬롯 A에서 뽑힌 그 액션이 넘어온 상태 | 2초 | 문장이 슬롯 A와 **글자까지 동일**해야 한다 |
| 2 | WBS | 담당·기한이 붙고 진척이 자동 계산 | 4초 | 상위 작업의 % 가 따라 올라가는 순간을 담을 것 |
| 3 | 대시보드 | 지연·위험 신호가 뜬 화면 | 3초 | 빨강/주황 신호가 화면에 있어야 "관리되고 있다"가 전달된다 |
| 4 | 주간보고 | **버튼 클릭 → 공식 양식 PPT 완성** | 4초 | 영상 전체의 절정. 클릭부터 완성까지 컷 없이 |

4번이 티저 15초의 소재이기도 하다. **가장 좋은 테이크를 여기에 쓴다.**

## 3. 합성

1. HTML 프레임을 61초 녹화 → 타임라인에 깐다
2. 슬롯 A 영상을 0:12.0 지점부터 얹고, 슬롯의 검은 사각형 영역(1920×1080 기준 좌표 **x 728~1305, y 224~980**)에 맞춰 스케일
3. 슬롯 B도 동일하게 0:29.0 지점부터
4. 슬롯 안의 촬영 지시 텍스트는 실화면에 가려져 사라진다 — 별도 삭제 불필요
5. 자막·캡션은 HTML이 이미 갖고 있으므로 편집기에서 추가하지 않는다

**음악**: 저작권 프리 트랙 하나. 0:00~0:06은 거의 무음에 가깝게 깔고, 0:11(슬롯 A 시작)에서 올린다.
**내레이션 없음** — IRIS 목록에서 소리 없이 재생될 가능성이 높아 자막만으로 완결되게 만들었다.

## 4. 15초 티저

`?teaser`를 붙여 열면 티저 모드로 뜬다. 슬롯은 하나뿐이고, 본편 슬롯 B의 4번(주간보고 클릭)을 재활용한다.
메신저로 돌릴 용도이므로 **세로 크롭(1080×1350) 버전도 함께 뽑는다** — 모바일에서 가로 영상은 손톱만 하게 보인다.

## 5. 촬영 전 최종 점검

- [ ] 화면 어디에도 실명·실제 거래처명·실제 금액이 없다 (2인 교차 확인)
- [ ] 슬롯 A와 B의 액션 문장이 글자까지 동일하다
- [ ] 주간보고 생성이 3초 안에 끝난다 (느리면 사전 워밍업 후 재촬영)
- [ ] 브라우저 확대율 100%, 다크모드 여부를 두 슬롯이 통일
- [ ] 61초 프레임 녹화에 마우스 커서가 안 잡혔다 (HTML 재생 중엔 커서를 화면 밖으로)

# WBS 웹 관리 도구 — 설계 문서

작성일: 2026-06-29
원본: `docs/WBS-original.xlsx` (D-CUBE PI Master Plan 수립 WBS)

## 1. 목적

엑셀로 관리하던 프로젝트 WBS(Work Breakdown Structure)를 웹 프로그램으로 전환한다. 핵심 동기 네 가지:

1. **여러 팀 동시 입력** — PMO/DT/ERP/MES가 각자 담당 작업 진척을 동시에 입력 (엑셀 파일 주고받기·버전 충돌 해소)
2. **진척/대시보드 한눈에** — 전체 진행률, 지연 작업, 팀별 현황 자동 집계
3. **간트 차트 시각화** — 주간 타임라인을 보기 좋게
4. **입력/편집 편의성** — 깔끔한 폼·표

## 2. 범위 & 제약

- **여러 프로젝트** 지원 (이번 WBS 외에도 향후 다른 프로젝트 WBS 생성 가능)
- **클라우드 배포** (Vercel + 호스팅 DB)
- **엑셀 자동 임포트** 지속 지원 (업로드로 WBS 가져오기)
- **진척 자동 계산** (담당자는 실적%만 입력)

## 3. 기술 스택 & 아키텍처

**스택: Next.js(App Router) + Supabase(Postgres + 인증 + RLS) on Vercel**

- 팀별 계정 + 권한 구분이 핵심 → Supabase 인증 + Row-Level Security(RLS)로 "각 팀은 자기 담당 작업만 편집"을 DB 레벨에서 강제
- Vercel 배포 궁합, 진척 집계 쿼리는 Postgres로 충분

```
브라우저(Next.js UI) → Next.js API/Server Actions → Supabase Postgres
                                                    ↳ 인증(팀 계정/역할)
                                                    ↳ RLS(권한 강제)
엑셀 업로드 → 파서(서버) → DB 시드/병합
진척% → DB 저장(실적만) → 서버에서 계획%·달성율·롤업 자동 계산
```

권한은 **RLS + 서버(API) 재검증 이중**으로 강제한다. UI 비활성화만 믿지 않는다.

## 4. 데이터 모델

```
Project           프로젝트 (여러 개 지원)
  id, name, start_date, end_date, created_at

WbsItem           WBS 항목 — 자기참조 트리 (Phase>Task>Activity 한 테이블)
  id, project_id, parent_id(최상위 null),
  level: 'phase' | 'task' | 'activity',
  code('1','1-1'…), sort_order, name, deliverable(선택),
  planned_start, planned_end,
  weight(leaf만; null이면 형제 균등),
  actual_pct(leaf만 입력, 0~100),
  status(자동 도출)

Team              PMO / DT / ERP / MES
  id, name

Membership        사용자 ↔ 팀 ↔ 역할
  user_id, team_id,
  role: 'pmo_admin'(전체 편집) | 'team_editor'(자기 담당만)

ItemOwner         항목별 담당 (엑셀의 ● / △)
  wbs_item_id, team_id, kind: 'primary'(●) | 'support'(△)

Holiday           프로젝트별 공휴일 (계획% 계산용)
  project_id, date, name

ChangeLog         변경 이력 (누가·언제·무엇을)
  user_id, wbs_item_id, field, old, new, at
```

**결정 사항:**

1. **단일 트리 테이블** — Phase/Task/Activity를 `WbsItem` 한 테이블에 `level`로 구분 (깊이 확장 유연)
2. **실적은 leaf(Activity)에만 입력** — 상위는 항상 자동 집계 (직접 입력 불가)
3. **담당은 별도 `ItemOwner` 행** — 한 작업에 여러 팀 ●/△ 가능
4. **변경 이력(ChangeLog) 포함**

## 5. 진척 자동 계산 로직

**① 계획% (leaf) — 일정 기준, 영업일·공휴일 반영**
```
계획% = (start ~ min(오늘,end) 영업일수) / (start ~ end 총 영업일수) × 100
        (주말 + Holiday 테이블 제외, 0~100 클램프)
```

**② 달성율 = 실적% ÷ 계획%** (계획% 0이면 '시작전' 표시)

**③ 상태 자동 도출**
- 실적 100% → **완료**
- 실적 < 계획 → **지연** (대시보드 빨강)
- 0 < 실적 ≤ 계획 → **진행중**
- 시작일 미도래 → **시작전**

**④ 롤업 (Task / Phase / 전체) — 가중 평균**
```
상위 실적% = Σ(leaf 가중치 × leaf 실적%) / Σ(leaf 가중치)
상위 계획% = Σ(leaf 가중치 × leaf 계획%) / Σ(leaf 가중치)
```
- **가중치 기본값**: 형제끼리 균등(1/n). PMO가 항목별로 덮어쓰면 그 값 사용.
- 엑셀의 가중치 칸이 비어 있으므로 초기엔 균등으로 시작.

## 6. 화면 구성

### ① WBS 보드 (메인) — 엑셀 대체 핵심 화면
- **좌측 트리 테이블**: Phase>Task>Activity 접기/펼치기. 상위 행은 자동 집계%(읽기전용), leaf는 실적% 인라인 입력.
- **권한 반영**: 팀 편집자는 자기 팀이 담당(●/△)인 행만 실적 입력 활성화, 나머지 회색. PMO는 전부 편집 + 일정/가중치/구조 수정.
- **우측 간트 차트**: 좌측 행과 정렬, 주간(W01~) 타임라인, 계획 바 + 실적 진행 음영, 주말·공휴일 음영, 오늘 세로선.
- 행 클릭 → 우측 상세 패널: 산출물 / 일정 / 담당(●△) / 실적% 입력 / 변경 이력.

### ② 대시보드 — "한눈에 진척"
- 상단 KPI: 전체 계획% vs 실적%, 달성율, 지연 작업 수
- Phase별 진행 막대(계획 vs 실적), 팀별(PMO/DT/ERP/MES) 진행률
- 지연 작업 Top 리스트(실적<계획), 이번 주 마감 예정 작업
- 산출물 현황(완료/예정)

### ③ 설정
- 프로젝트 목록·생성·전환
- 멤버/팀/역할 관리
- 공휴일 관리
- 엑셀 임포트(업로드) / 익스포트(다운로드)

### ④ 로그인 — 팀 계정 로그인

## 7. 기능 우선순위 (단계적 빌드)

| 단계 | 범위 |
|------|------|
| **MVP** | 로그인·권한, WBS 트리 CRUD, 실적 입력, 진척 자동계산, **간트 차트**, 엑셀 임포트, 기본 대시보드 |
| **2차** | 변경 이력, 엑셀 익스포트 |
| **3차** | 지연 알림, 산출물 첨부, 주간 스냅샷/추세 |

> 간트 차트는 핵심 시각화이므로 MVP에 포함한다.

## 8. 에러·예외 처리

- **동시 편집**: 같은 행 동시 저장 시 마지막 저장 반영 + 변경이력 기록(낙관적 잠금). 충돌 시 토스트 안내.
- **엑셀 임포트**: 구조 불일치 시 행 단위 검증 리포트("3행 날짜 오류") 후 부분 반영 거부. 원본 구조(`docs/WBS-original.xlsx`: A=Biz, B=Phase, C=Task, D=Activity, G~J=담당 PMO/DT/ERP/MES, L=산출물, M/N=계획 Start/End, S~ 간트) 기준 파서.
- **권한**: 서버(API)에서도 재검증 — RLS + 서버 체크 이중.

## 9. 테스트 전략

- **진척 계산 로직**(계획%·달성율·롤업·영업일 제외) — 단위 테스트 필수
- **권한별 접근**(PMO vs 팀 편집자) — 통합 테스트
- **엑셀 임포트 파서** — 정상/오류 케이스 단위 테스트

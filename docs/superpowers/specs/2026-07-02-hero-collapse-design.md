# 히어로 섹션 접기/펼치기 설계

- 날짜: 2026-07-02
- 상태: 사용자 부재 중 추천 기본값으로 확정(아래 "열린 선택지" 참조) — 사후 조정 가능
- 요청: "해당 프로젝트의 히어로 섹션 부분을 접었다 펼쳤다 할 수 있는 기능을 추가하고 싶어"

## 배경

프로젝트 하위 6개 페이지(대시보드·WBS·칸반·멤버·근태·설정)는 모두 공용 컴포넌트
`src/components/ui/PageHero.tsx`(서버 컴포넌트)로 다크 히어로 카드를 렌더한다.
히어로에는 제목·설명·KPI 카드 행(`heroKpis`)·액션 버튼이 들어가 세로 공간을 크게 차지하므로,
WBS 시트나 칸반처럼 작업 영역이 중요한 화면에서 접을 수 있으면 유용하다.

## 검토한 접근안

| | 접근 | 트레이드오프 |
|---|---|---|
| **A (채택)** | `PageHero`를 클라이언트 컴포넌트로 전환, 내부 상태로 접기 | 한 파일 수정으로 6개 페이지 전부 적용. ReactNode props는 서버에서 직렬화되어 전달되므로 데이터 페칭은 서버에 유지. |
| B | 서버 `PageHero`를 감싸는 클라이언트 래퍼 `<CollapsibleHero>` | 파일 추가 + 접힘 시 내부 스타일(제목 크기·패딩) 조정이 어려움. 이점 없음. |
| C | URL searchParams로 서버 주도 토글 | 깜빡임 없음이 장점이나, 순수 시각 토글에 내비게이션·리페치 발생. 과함. |

## 설계 (접근안 A)

### 동작

- 히어로 카드 우상단에 아이콘 토글 버튼(ChevronUp/ChevronDown, lucide) 추가.
- **펼침(기본):** 현재와 동일 — 배지·아이브로우·제목·설명·액션·KPI 행 모두 표시.
- **접힘:** 컴팩트 바 — 제목 한 줄(truncate, 작은 크기) + 토글 버튼만. 배지·아이브로우·설명·액션·KPI·aside 숨김. 패딩 축소(`px-6 py-4`).
- 접힘/펼침 시 제목 아래 영역은 CSS `grid-template-rows 1fr↔0fr` 트랜지션으로 부드럽게 애니메이션.
  배지·아이브로우(1줄)는 조건부 렌더로 단순 처리. `prefers-reduced-motion`은 전역 CSS가 이미 처리.

### 상태·지속성

- `localStorage('dflow-hero') === '1'` — 사이드바(`dflow-sidebar`)와 동일 컨벤션.
- 키는 **전역 1개**(페이지별 아님): 사이드바 컨벤션과 일치, "작업 모드"로 쓰는 사용자 멘탈 모델에 부합.
- **모듈 레벨 캐시**(리뷰 반영): PageHero는 페이지 트리에 있어 소프트 내비게이션마다 리마운트되므로,
  사이드바처럼 useEffect로만 읽으면 페이지 이동마다 펼침→접힘 플래시가 재생된다.
  모듈 변수에 상태를 캐시해 리마운트 시 useState 초기값으로 사용 → 이동 시 플래시 0회.
- 하드 로드 1회에 한해 SSR 펼침 → 접힘 전환이 남음(사이드바의 하드 로드 플래시와 동일 빈도로 수용).

### 모션 (리뷰 반영)

- 모든 트랜지션 duration-200 — 사이드바(transition-[width] duration-200)와 통일.
- 수평 패딩은 접힘/펼침 동일(px-6 sm:px-8) 유지, 수직 패딩만 변경 + `transition-[padding]`.
- 제목 폰트 크기(34→18px)·마진도 `transition-[font-size,margin]`으로 함께 애니메이션.

### 접근성·i18n

- 버튼에 `aria-expanded`, `aria-controls`, `aria-label`은 로케일 사전 키(`ui.heroExpand`/`ui.heroCollapse`) 사용.
- 접힌 콘텐츠는 `visibility: hidden` 처리로 접근성 트리·탭 순서에서 제외.
- 클리핑 래퍼(overflow-hidden)에 `-mx-1 px-1` 여유 — 전역 `:focus-visible` 아웃라인(2px+offset 2px)이 좌측 끝 액션 버튼에서 잘리던 회귀 수정(리뷰 반영).

### 범위 제외

- 프로젝트 목록(홈) 페이지의 히어로(별도 마크업) — "해당 프로젝트" 요청 범위 밖.
- 서버 지속성(DB 저장), 페이지별 상태 분리, no-flash 인라인 스크립트.

## 열린 선택지 (사용자 확인 필요 시 조정)

1. 적용 범위: 전체 6개 페이지(채택) vs 대시보드만 vs 작업 화면만 → 공용 컴포넌트라 어느 쪽이든 쉽게 좁힐 수 있음.
2. 상태 단위: 전역 1키(채택) vs 페이지별 키(`dflow-hero:{page}`).
3. 접힘 시 잔존 요소: 제목만(채택) vs 제목+KPI 요약.

## 테스트·검증

- `npm run build` + `npm run lint` + `npm run test`(vitest) 통과.
- 실환경 브라우저 접근 불가 세션이므로 빌드/린트/테스트 기반 검증(프로젝트 컨벤션).
- 3렌즈(정확성/디자인 일관성/접근성) 병렬 리뷰 + 발견별 적대적 검증 워크플로우 실행:
  발견 8건 중 4건 확정(내비게이션 플래시 ×2, 포커스 링 클리핑, 모션 불일치) → 전부 수정 반영.
  기각 4건은 미사용 aside 경로·이론적 aria-controls 우려로 실사용 영향 없음 판정.

# 바이브코딩 경진대회 우승 패키지 — 설계 문서

- 작성일: 2026-07-11
- 상태: 사용자 승인 완료 (섹션 1~4 개별 승인)
- 목표 시점: 2026년 9월 사내 바이브코딩 경진대회 (우승 시 2027년 1월 CES 참관)

## 1. 배경과 목표

D'Flow는 동국씨엠 PI(Process Innovation) 관리를 위해 솔로 바이브코딩으로 구축한
운영 스위트로, 현재 팀이 실제 업무에 사용 중이다. 9월 사내 경진대회(임원급 심사,
라이브 데모 + 발표)에서 우승하기 위해 무엇을 더 만들지를 이 문서로 확정한다.

### 대회 컨텍스트 (클래리파잉 결과)

| 항목 | 확인된 사실 |
|------|------------|
| 대회 성격 | 사내 대회, 임원급 심사 |
| 발표 형식 | 라이브 데모 + 발표 |
| 실사용 | 팀이 실제 업무에 사용 중 (실데이터·실사용자 보유) |
| 제작자 포지션 | IT/개발 유관 직무 — "AI로 생산성을 극대화한 개발자" 서사 |
| AI 예산 | 계속 무료 티어만 (유료 API 불가) |
| 투입 가능량 | 시간 여유 있음 — 크게 간다 |
| 경쟁 구도 | 강력한 경쟁자 존재 — 최고 수준 가정 |

### 핵심 서사

**"기록하는 도구에서 일하는 동료로."** 철강회사 PI 현장에서 실제로 쓰이는 운영
스위트가, 이제 말을 알아듣고 스스로 일한다.

### 현황 진단 (8개 서브시스템 병렬 분석 + 완결성 크리틱 요약)

강점: ① 엑셀→살아있는 WBS 왕복 ② 구글시트급 실시간 동시편집(셀 프레즌스)
③ 원클릭 사내 공식 양식 PPT ④ 환각 없는 무료 티어 RAG 챗봇(429 3중 방어)
⑤ 엔지니어링 규율(테스트 662개, RLS+트리거 이중 방어, 보상 트랜잭션).

약점: ① 챗봇이 읽기 전용 Q&A ② 대시보드가 보고에서 끝남(예측·시뮬레이션 부재)
③ 데모 지뢰(마크다운 별표 노출, 스트리밍 출처 미표시, 토글 숨김, 재임포트 중복,
RPM 20 강등, 스냅샷 이력 부족) ④ 하드코딩 범용성 ⑤ 리포 위생(실명 파일 커밋,
README 드리프트).

대회 자산 발견: 전체 365커밋이 2026-06-25 이후(약 2.5주 솔로 빌드),
설계 문서 17개·구현 계획 12개·멀티에이전트 QA 산출물이 리포에 보존 —
바이브코딩 프로세스의 증거로 활용한다.

## 2. 5분 라이브 데모 아크 (개발 우선순위의 기준)

1. **오프닝 (30초)** — 실제 양식의 (익명화된) 엑셀 업로드 → WBS 트리 원자 생성.
2. **협업 증명 (30초)** — 브라우저 2개, 주간 시트 셀 단위 커서 + 실시간 반영.
3. **센터피스 (2분)** — 음성으로 "ERP 인터페이스 설계 실적 80으로 올려줘" →
   확인 카드 → 적용 → 시트·롤업·대시보드 연쇄 갱신. 회의록 업로드 →
   액션아이템 자동 추출 → 원클릭 WBS 반영.
4. **ROI 클라이맥스 (1분)** — "주간보고 초안 만들어줘" → AI 초안 →
   회사 공식 양식 PPT. "주간보고 30분 → 3분."
5. **임원의 순간 (1분)** — What-if: "이 Phase 2주 지연되면?" →
   예상 준공일·S-Curve·신호등 즉시 재계산. 엔딩은 /story 페이지 —
   "이 시스템은 자기가 어떻게 만들어졌는지도 압니다."

## 3. Track A — Agentic AI (센터피스)

### A1. 액션 봇: 자연어 명령 → 확인 카드 → 기존 서버 액션

- **명령 파싱**: 기존 의도 분류(src/lib/ai/intent.ts)에 "명령" 경로 추가.
  LLM JSON 단발 호출로 발화를 `{action, target, params}`로 추출.
  지원 액션 카탈로그 v1은 **실적 변경 · 일정 변경 · 완료 처리** 3종으로 한정.
  스트리밍 function calling 대신 JSON 모드 — 기존 llm.ts 모델 체인·429 방어 재사용.
- **대상 해석은 결정형**: LLM이 아니라 팩트시트(buildFactSheet) 기반
  정규화 부분일치 매칭. 후보 다수 → 후보 칩으로 되묻기, 0건 → 정직하게 "못 찾았어요".
- **확인 카드 필수 (자동 실행 없음)**: 봇 말풍선 안에 변경 요약
  (항목명, 필드, 이전값 → 새값)과 [적용]/[취소] 버튼. 적용 시에만 기존 서버 액션
  (updateActual, updateWbsFields 등) 호출 — 권한 게이트·낙관적 잠금·change_logs·
  진척 스냅샷이 전부 그대로 적용된다. 권한 없는 계정의 명령은 기존 에러가
  그대로 노출된다("AI도 권한을 우회하지 못한다"가 데모 포인트).

### A2. 음성 입력

- DkBot 입력창에 push-to-talk 마이크 버튼. 브라우저 내장 Web Speech API(ko-KR),
  중간 인식 결과를 입력창에 실시간 표시. 비용·서버 0.
- 크롬 전용으로 충분(데모 환경 크롬). API 미지원 시 버튼 자체를 숨긴다.
  음성 권한 거부 시 토스트 안내 후 키보드 입력 유지.

### A3. 회의록 → 액션아이템 → WBS 반영

- 회의록 뷰어에 "액션아이템 추출" 버튼. LLM JSON 모드로
  `{결정사항[], 액션아이템[{제목, 담당?, 기한?, 연관작업?}]}` 추출.
- 추출 결과는 카드 목록으로 표시, 각 카드에 "WBS에 추가"(기존 하위 추가 액션
  재사용, 부모 선택 UI). 연관작업 후보는 팩트시트 매칭으로 제안.
- 추출 결과는 신규 테이블(마이그레이션 0025, minute_action_items)에 저장 —
  제안됨/반영됨/무시 상태 추적. "회의가 작업이 되는" 흐름이 이력으로 남는다.

### A4. 주간보고 AI 초안

- 주간 시트에 "AI 초안" 버튼. 서버가 **결정형 재료**(해당 주 change_logs,
  완료/진행 항목, 해당 주 회의록, 차주 예정 항목)를 수집 → LLM이 모듈별
  금주실적/차주계획 문안 작성.
- **빈 셀만 채운다. 사용자 내용은 절대 덮지 않는다.** 초안 셀은 시각적 마커로 구분.
- LLM 실패 시 결정형 불릿 초안(사실 나열)으로 폴백. 이후 기존 PPT 파이프라인 연결.
- ROI 서사: "주간보고 작성 30분 → 3분" 정량 주장의 근거 기능.

### 레이트리밋 전략

A1/A3/A4 모두 상호작용당 LLM 1콜(JSON 단발) — 데모 중 호출 수를 예측·통제
가능. 리허설로 RPM 20 내 페이스 확정.

## 4. Track B — What-if 시뮬레이터

- **위치**: 대시보드 "시뮬레이션" 패널. Phase(또는 서브트리) 선택 +
  슬라이더/±버튼으로 "N일 지연/단축".
- **출력**: 예상 준공일·신호등 판정·S-Curve 계획선을 현재 vs 시뮬레이션
  오버레이로 즉시 재계산.
- **구현**: 순수 도메인 함수(computeTree, plannedAt, buildExecSummary)를
  클라이언트에서 재실행(src/lib/domain/simulate.ts 신설). 서버 왕복·데이터 변경
  없는 100% 샌드박스.
- **정직한 한계 명시**: 선후행 의존관계 모델이 없으므로 "선택 서브트리의
  단순 일정 이동" 시뮬레이션임을 UI에 표기. 의존관계 전파는 범위 제외.

  않는지 확인 필요 (실화면에서 로컬 다크 강제가 계정 동기화로 롤백됨 — 설계
  의도대로지만 토글 UX 검증 필수).

### 의도적으로 하지 않는 것 (감사 지적됐으나 대회 전 제외)

i18n 전면 정비, 다크모드 전용 이슈 묶음(토글 숨김 상태), 죽은 코드 정리,
접근성 심화(포커스 트랩 추가 등 — 급하게 넣으면 신규 버그 위험), 타이포
스케일 토큰화(178곳 일괄 치환은 시각 회귀 위험), 모바일/태블릿 대응(데스크탑
+프로젝터 데모 기준), 챗봇 컴포넌트 공용화 리팩터링. 근거: 데모 노출 확률
대비 회귀 위험.

## 6. 아키텍처 원칙

- **새 인프라 금지, 기존 자산 재사용**: 명령 파싱·대상 매칭·초안 재료 수집·
  시뮬레이터 수학은 순수 함수 모듈(src/lib/ai/commands/, src/lib/domain/simulate.ts)
  — 기존 도메인/표현 분리 관례 유지.
- 모든 LLM 호출은 기존 llm.ts 체인(429 3중 방어·모델 폴백) 경유, JSON 단발 모드.
- 새 외부 의존성 0개(음성은 브라우저 내장 API).
- 스키마 추가는 0025(minute_action_items) 1개뿐.
- 쓰기는 전부 기존 서버 액션 경유 — AI 경로 전용 쓰기 우회를 만들지 않는다.

## 7. 에러 처리 (기존 "정직한 강등" 철학 유지)

| 상황 | 처리 |
|------|------|
| 명령 파싱 실패 | 예시 문장 안내 |
| 대상 모호 (후보 다수) | 후보 칩으로 되묻기 |
| 대상 0건 | "못 찾았어요" + 검색어 제안 |
| 권한 없음/편집 충돌 | 기존 한국어 에러를 봇 말풍선에 그대로 표출 |
| LLM 불능 (명령) | "지금은 명령 처리가 어려워요" + 수동 UI 안내 |
| LLM 불능 (주간초안) | 결정형 불릿 초안 폴백 |
| LLM 불능 (액션아이템) | 에러 카드 + 재시도 버튼 |
| 음성 권한 거부/무음 | 토스트 안내, 키보드 입력 유지 |

## 8. 테스트 전략

- 순수 함수(명령 카탈로그 검증, 대상 매처, 시뮬레이터 수학, 초안 재료 빌더):
  vitest 단위 테스트 — 기존 tests/ 스타일.
- 서버 액션 권한 게이트: 기존 accounts-gate 패턴의 모킹 테스트
  (게이트 통과 전 쓰기 도달 금지 스파이).
- 런타임 검증: verify 스킬(빌드/린트/테스트 + curl), 프로덕션 배포 후 실사용 확인.
- 데모 리허설 체크리스트 문서를 7주차에 작성, 2회 리허설.

## 9. 8주 일정

| 주차 | 내용 |
|------|------|
| 1–2주 | A1 액션 봇 (파싱→매칭→확인 카드→3종 액션) + C1 리포 위생·README |
| 3주 | A2 음성 입력 + C2 DK Bot 데모 품질 (마크다운·출처·토글) + 디자인 S 수정 7건 |
| 4주 | A3 회의록→액션아이템 (0025 마이그레이션) |
| 5주 | A4 주간보고 AI 초안 |
| 6주 | Track B What-if + 스냅샷 시드 + 엑셀 교체 임포트 + 디자인 M 수정(칸반 낙관 업데이트·시트/타임라인 토글) |
| 7주 | /story 페이지 + 데모 리허설 1차 + 잔여 하드닝(회의록 챗 패널 패키지·폰트 셀프호스팅) |
| 8주 | 버퍼 + 리허설 2차 + 발표 준비 |

완료 기준: 각 기능은 프로덕션 배포 + 팀 실사용까지. "대회용 기능"이 아니라
"실사용 증거"를 쌓는다.

## 10. 범위 제외 (Non-goals)

- 선후행 의존관계 모델·크리티컬 패스 (데이터 모델 신설 필요, What-if는 단순
  일정 이동으로 충분)
- 권한 모델 프로젝트 스코프 개편 (실사용 운영 리스크, Q&A 대비로 갈음)
- WBS 시트 가상화, plannedAt 캐시 (데모 규모에서 비문제)
- 유료 API·외부 STT (무료 티어 제약 유지)
- i18n 전체 완역 (기존 보류 결정 유지 — 데모는 한국어 진행)
- 명령 카탈로그 확장 (회의 생성·공지 작성·담당 변경 등은 v1 이후)

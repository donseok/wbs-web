# 주간보고 점검 — 번호 표기 통일 (설계)

2026-07-24 · 사용자 승인된 방향: 점검(주간보고 린트)에 "번호 표기 통일"을 추가한다.
별도 버튼이 아니라 기존 점검 규칙의 확장이다(과거 "양식 통일" 버튼은 사용자 결정으로 제거된 이력이 있음).

## 목표

작성자마다 다른 번호 표기(`1.` vs `1)`, 번호 뒤 공백 유무)를 시트 전체 기준으로 통일해
PPT 생성 결과가 깔끔해지게 한다.

## 규칙 변경 — 체번(numbering) 규칙 확장

새 kind를 만들지 않는다. 기존 `lintNumbering`이 "셀의 번호 줄"을 이미 소유하므로,
같은 규칙이 **순서(체번) + 표기(스타일)** 를 함께 본다. 두 규칙이 같은 줄을 서로 다르게
고쳐 편집이 충돌하는 구조를 원천 차단하고, 셀당 지적 1건·적용 1번으로 끝낸다.

1. **기준 표기(구분자)** — 시트 전체의 번호 줄을 세어 `.`/`)` 다수결. 동수면 `.` 우선.
   시트에 한 종류만 있으면 그 표기를 존중한다(글머리 기호 규칙과 같은 정신).
   구분 단위 원칙의 예외가 하나 늘어난다: 글머리 기호에 이어 번호 표기도 시트 전체 기준
   (보고서 겉모습 문제이므로). 파일 머리 주석·패널 안내 문구를 함께 갱신한다.
2. **번호 뒤 공백** — 항상 정확히 1칸으로 정규화(`1.가` → `1. 가`, `1.  가` → `1. 가`,
   전각 공백 gap 포함). 이는 번호 "표기"의 일부이며, 제외 결정된 본문 공백 검사
   (줄 끝·연속·전각 공백, 빈 줄)와는 별개다 — 본문 쪽은 여전히 건드리지 않는다.
3. **체번** — 기존과 동일하게 번호 줄 2개 이상 + 1..n 이 아닐 때만 재부여하되,
   재부여 시 "첫 줄 표기 상속" 대신 위의 기준 표기를 쓴다.
4. **지적 발화 조건** — 셀의 번호 줄을 기준 표기·공백·(필요 시) 재부여로 고친 결과가
   원문과 다르면 지적 1건. 표기만 어긋난 번호 줄 1개짜리 셀도 지적된다(신규 동작).
   detail은 바뀌는 내용별로 밝힌다: 체번(`줄 번호가 1, 2, 4 입니다 → 1, 2, 3`),
   표기(`번호 표기 → '1.' (시트 전체 기준)`), 공백(`번호 뒤 공백 → 1칸`).
   id는 `numbering:{rowId}:{cellKey}` 그대로(안정 키 유지).

## 번호 줄 판정 가드 (공유 술어, 잠복 버그 동시 수정)

현재 `NUM_PREFIX(/^(\d+)([.)])( *)/)`는 날짜·소수를 목록 번호로 오인한다.
`normalizeForCompare`가 `1.5배 개선`/`2.5배 개선`을 같은 줄로 접어 **완전 중복 자동 삭제**로
데이터가 지워질 수 있고, 체번이 `2026.` 를 번호로 세어 날짜를 재부여할 수 있다.

새 공유 술어 `listNumberLine`(이름은 구현에서 확정)로 교체한다. 목록 번호 줄로 인정하는 조건:

- 선두(들여쓰기 제외)가 `숫자 1~2자리 + (. 또는 )) + 공백(반각·전각) 0개 이상`
- 뒤에 본문이 있어야 한다(`1.` 단독 줄은 제외 — 공백을 덧붙이는 편집을 만들지 않기 위해)
- **공백 0개이면서 본문 첫 글자가 숫자면 제외** — `2026.07.24`(날짜), `1.5배`(소수),
  `1.2 개요`(절 번호)를 보호한다. 자리수 상한(2자리)도 같은 목적의 이중 안전장치.

이 술어를 세 곳이 공유한다: 다수결 집계, 체번+표기 수정, `normalizeForCompare`의 번호 떼기.
공유하지 않으면 "집계엔 세는데 수정에선 빠지는" 어긋남이 생긴다(글머리 규칙에서 이미 배운 것).

부수 효과(의도됨): `normalizeForCompare`가 날짜·소수 선두를 더는 떼지 않으므로
중복/유사중복 오탐이 줄어든다. 3자리 이상 번호(`100. 항목`)는 떼지 않게 되지만,
같은 표기끼리는 여전히 동일 문자열이라 중복 검출에 실질 영향 없음.

## 바뀌지 않는 것 / 비범위

- duplicate·nearDuplicate·format(글머리) 규칙의 판정 로직, KIND 구성, 패널 UI 구조.
- 개행 패턴 통일(항목 사이 빈 줄 등) — 이번 범위에서 제외(사용자와 합의).
- 중첩 번호 목록의 계층 인식(기존 평면 재부여 의미 유지), 전각 구분자(`１．` 등).
- 기존 테스트 중 "구분자 뒤 공백을 보존한다"는 이번 정책 변경으로 의도적으로 대체된다.
